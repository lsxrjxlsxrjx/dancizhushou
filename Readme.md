### 第一阶段：在 GitHub 建立“云端数据库” (建库)

我们需要在 GitHub 创建一个专门用来存放词库数据的私人仓库。

1. **新建私有仓库**：
* 登录 GitHub 主页，点击右上角的 **“+”** 号，选择 **New repository**。
* **Repository name**（仓库名称）：起个名字，例如 `my-word`。
* **Visibility**（可见性）：**务必选择 Private（私有）**，确保只有你自己能看。
* **Initialize**（初始化）：**勾选 "Add a README file"**（这一步必须勾选，用于初始化主分支）。
* 点击底部的 **Create repository**。


2. **创建数据占位文件**：
* 进入刚建好的仓库页面，点击绿色的 **Add file** 按钮，选择 **Create new file**。
* 在文件名输入框中输入：`vocab_data.json`。
* 在下方大文本框中输入初始的空数据结构：
```json
{
  "words": [],
  "config": {}
}

```


* 点击右上角绿色的 **Commit changes...** 按钮，直接提交保存。

### 第二阶段：生成专属门禁卡 (配钥匙)

为了让你的网页能安全地修改这个私有仓库，我们需要生成一个仅对该仓库拥有读写权限的细粒度 Token（Fine-grained PAT）。

1. **进入开发者设置**：
* 点击 GitHub 网页右上角你的头像，选择 **Settings**（设置）。
* 在左侧菜单滑到最底下，点击 **Developer settings**。


2. **生成细粒度 Token**：
* 在左侧菜单选择 **Personal access tokens** -> **Fine-grained tokens**。
* 点击右上角 **Generate new token**。
* **Token name**：随便填，例如 `word-app-token`。
* **Expiration**：选择过期时间（建议选最长的 1 年，过期后按此流程重新生成即可）。


3. **绑定仓库 (核心步骤)**：
* 在 **Repository access** 选项中，选择 **Only select repositories**。
* 在下拉列表中，找到并选中你刚刚创建的仓库（如 `用户名/my-word`）。


4. **授予读写权限 (核心步骤)**：
* 展开下方的 **Repository permissions** 菜单。
* 找到 **Contents** 这一项，将右侧下拉菜单从 `No access` 改为 **Read and write**。


5. **保存并复制 Token**：
* 滑到页面最底部，点击绿色的 **Generate token**。
* ⚠️ **极度重要**：页面跳转后，屏幕上会出现以 `github_pat_` 开头的一长串字符。**请立刻复制并妥善保存在你的电脑记事本里**，刷新页面后它将永远消失！

### 第三阶段：在小程序中配置连接 (绑程序)

现在把“数据库地址”和“钥匙”交给你的背单词网页。

1. 打开你本地或部署好的“背鸡毛单词”网页。
2. 点击右上角的 **⚙ 设置**。
3. 找到 **☁️ 云端同步设置 (GitHub 官方仓库 API)** 区域：
* **GitHub 仓库名**：填入你的仓库全称。格式必须是 `你的GitHub用户名/仓库名`（例如：`lsxrjxlsxrjx/my-word`）。
* **专属 Token**：把刚刚复制的 `github_pat_...` 完整粘贴进去。


4. 点击底部的 **💾 保存设置**。

---

### 第四阶段：日常使用指南

配置完成后，点击页面顶部的 **☁️ 云同步** 按钮即可呼出同步面板。

* **📤 覆盖上传到云端 (Upload)**
* **使用场景**：当你在网页上背了新单词、修改了释义、或者改变了学习进度后。
* **逻辑原理**：程序会将你浏览器里的所有最新数据，打包成 Base64 格式，直接推送到 GitHub，**完全覆盖**云端 `vocab_data.json` 的旧内容。
* **习惯建议**：每次结束学习准备关掉网页前，点一次上传。


* **📥 从云端下载数据 (Download)**
* **使用场景**：当你在另一台电脑/手机上打开网页，想要获取最新的学习进度；或者你本地浏览器缓存被清空时。
* **逻辑原理**：程序会向 GitHub 发起请求，把云端完整的 JSON 数据拉取下来，**强制覆盖**当前浏览器里的所有数据。
* **习惯建议**：如果你在多台设备间切换背单词，每次在新设备打开网页时，**第一件事就是先点下载**，确保拿到最新进度再开始背词，防止数据产生冲突。
